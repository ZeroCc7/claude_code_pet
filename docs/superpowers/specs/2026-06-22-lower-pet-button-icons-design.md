# 首页宠物位置与按钮图标设计

## 目标

- 在当前 `codex/codex-lifecycle-hooks` 首页布局基础上，将宠物整体下移 14px。
- 宠物脚部停在 `y=114` 属性栏上方，不覆盖属性和按钮区域。
- 用正式美术素材替换底部四枚程序绘制占位图标。

## 宠物布局

- 宠物合成区域保持 `x=28`、`y=14`、`72×100`，避免扩大刷新区域。
- 宠物基准位置从区域内 `y=19` 调整为 `y=33`，即屏幕坐标从约 `y=33` 下移到约 `y=47`。
- 互动弹跳仍相对新基准位置向上运动。
- 幼年宠物与最终人形共用同一基准下移量。
- 属性栏继续从 `y=114` 开始，底部按钮区继续从 `y=130` 开始。

## 四枚按钮图标

图标采用统一的中国古风修仙像素 UI：

| 按键 | 功能 | 图案 | 主色 |
|---|---|---|---|
| K1 | 互动 | 手掌托起一颗灵光 | 鎏金、暖白 |
| K2 | 培养 | 盛开的莲花 | 粉玉、鎏金 |
| K3 | 历练 | 山峰与向上小径 | 青玉、淡青 |
| K4 | 状态 | 展开的古卷轴 | 暖白、鎏金 |

素材要求：

- 原始生成使用严格 2×2 网格。
- 每格只放一个图标，不含文字、数字、边框和投影。
- 纯洋红色 `#FF00FF` 背景，便于去背。
- 最终固件图标统一为 `14×14` RGB565＋1-bit mask。
- 图标在 32px 宽按钮单元中水平居中。
- 保留底部四列分隔线，不重新加入 K1～K4 文字。

## 素材与固件

- 原始图：`assets/raw/ui/button_icons/`
- 处理图：`assets/processed/ui/button_icons/`
- 固件头文件：`firmware/ai_pet/assets/home_button_icons.h`
- 转换脚本：`scripts/convert_button_icons.py`
- `game_ui.cpp` 使用 `drawButtonIcon()` 绘制四枚图标。

## 验收

- 首页宠物比当前版本下移 14px。
- 宠物最低可见像素不覆盖 `y=114` 属性栏。
- 四枚图标在 128×160 实机上含义可辨、风格统一。
- 动态刷新不重新引入全屏闪烁。
- Python UI 测试和 Arduino 固件编译通过。
