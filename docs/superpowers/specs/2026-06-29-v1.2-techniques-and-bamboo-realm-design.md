# V1.2 功法修炼与青竹灵境设计

## 1. 目标

V1.2 解决两个问题：

- 养成内容偏少，玩家除了 AI 任务、升级和刷青云山道之外缺少长期成长目标。
- 自动战斗偏轻松，后续区域需要更明显的难度阶梯。

V1.2 的主题是“功法修炼 + 新关卡 + 战斗压力”。功法提供长期养成和战斗反制路径；青竹灵境提供新的推进目标；Boss 数值和区域配置提高压力，让功法成长有实际意义。

V1.2 不改变首页四键布局、不改变 AI Hook 协议、不改变进化分支判定，不引入装备、稀有度、合成或主动技能系统。

## 2. 功法系统

### 2.1 功法列表

新增四门功法，对应现有四维倾向：

| 功法 | 对应倾向 | 定位 |
|---|---|---|
| 太虚剑诀 | 剑 | 缩短战斗，提高伤害、暴击和破甲 |
| 九转丹法 | 丹 | 改善资源循环，提高道具收益和恢复效率 |
| 不灭玄功 | 体 | 提高容错，降低受伤并提供护盾 |
| 万灵息法 | 灵 | 提高续航，增加回血、闪避和灵力上限 |

每门功法等级为 `Lv0` 到 `Lv9`：

- `Lv0`：未修。
- `Lv1-Lv3`：入门。
- `Lv4-Lv6`：小成。
- `Lv7-Lv9`：大成。

功法全部自动生效，不需要装备、主修选择或战前切换。功法等级不参与 LV3/LV12 进化分支判定，进化仍只看四维倾向。

### 2.2 升级规则

功法升级需要同时满足：

- 对应倾向达到当前目标等级的门槛。
- 灵石足够支付升级消耗。

升级只消耗灵石，不消耗倾向。四门功法共用同一套门槛和消耗曲线：

| 目标等级 | 倾向门槛 | 灵石消耗 |
|---:|---:|---:|
| Lv1 | 0 | 50 |
| Lv2 | 10 | 90 |
| Lv3 | 22 | 150 |
| Lv4 | 38 | 240 |
| Lv5 | 58 | 360 |
| Lv6 | 82 | 520 |
| Lv7 | 112 | 720 |
| Lv8 | 148 | 960 |
| Lv9 | 190 | 1250 |

该曲线偏慢，设计为长期目标。入门应较快出现，小成需要稳定使用对应 AI 工具或刷对应区域，大成不要求在 V1.2 当天毕业。

### 2.3 功法效果

#### 太虚剑诀

| 等级 | 新增效果 |
|---:|---|
| Lv1 | 造成伤害 +3% |
| Lv2 | 暴击率 +2% |
| Lv3 | 造成伤害再 +3% |
| Lv4 | 破甲 +1 |
| Lv5 | 造成伤害再 +4% |
| Lv6 | 暴击率再 +3% |
| Lv7 | 造成伤害再 +5% |
| Lv8 | 破甲再 +1 |
| Lv9 | 暴击伤害 +20% |

破甲在玩家本回合最终伤害计算后生效。每 1 点破甲使 Boss 额外受到 2 点固定伤害，最低不改变原本 1 点伤害下限，不引入主动操作。

#### 九转丹法

| 等级 | 新增效果 |
|---:|---|
| Lv1 | 灵草恢复灵力 +1 |
| Lv2 | 回春丹恢复体力 +5 |
| Lv3 | 通关灵石奖励 +5% |
| Lv4 | Boss 开战最低灵力需求 -1 |
| Lv5 | 通关灵石奖励再 +5% |
| Lv6 | 灵草恢复灵力再 +1 |
| Lv7 | 回春丹恢复体力再 +5 |
| Lv8 | 通关灵石奖励再 +5% |
| Lv9 | 灵力与体力被动恢复间隔从 5 分钟缩短到 3 分钟 |

九转丹法 Lv9 只影响设备运行期间的被动恢复，不补算关机时间。

#### 不灭玄功

| 等级 | 新增效果 |
|---:|---|
| Lv1 | 受到伤害 -3% |
| Lv2 | 战斗开始获得 3 点护盾 |
| Lv3 | 战败后体力恢复值 +5 |
| Lv4 | 受到伤害再 -4% |
| Lv5 | 战斗开始护盾再 +4 |

护盾只在本场 Boss 战中吸收伤害，战斗结束后清零，不进入背包或通用资源。

#### 万灵息法

| Lv9 | 灵力上限 +30%，至少 +10 |

## 4. 秘境令改造

V1.1 的信物效果不够清晰。V1.2 将信物改为通用消耗品“秘境令”。

秘境令作用：

- 不绑定区域。
- 入境前可选择消耗 1 枚。
- 消耗后直接开启当前区域 Boss 挑战。
- 战斗胜利、失败、撤退或灵力耗尽都不返还。

流程：

```text
秘境选择
  ↓ K1 进入区域
如果持有秘境令且灵力足够
  ↓
┌──────────────┐
│    秘境令     │
├──────────────┤
│ 消耗 1 枚令牌  │
│ 直接挑战首领？ │
│ 持有：3        │
├──────────────┤
│ K1使用  K4不用 │
└──────────────┘
```

选择结果：

- K1 使用：消耗 1 个秘境令，当前区域 Boss 解锁，进入 Boss 战前准备页。
- K4 不用：进入普通历练页。
- 无秘境令或灵力不足：直接进入普通历练页。

受伤修士事件成功时，V1.2 改为获得 1 个秘境令，不再自动解锁当前 Boss。

背包第五个物品显示为“秘境令”，说明文案为“入境前使用，可直接挑战当前秘境首领”。

## 5. 青竹灵境

V1.2 新增并开放第二个区域“青竹灵境”。它复用青云山道的主流程和四类事件，不新增事件类型。

复用流程：

- 自动前进。
- 四类事件：采集、妖兽、伤者、捷径。
- 进度到 100% 后解锁 Boss。
- 可在入境前消耗秘境令直接挑战 Boss。
- 自动 Boss 战。
- 通关奖励、轮回、宝物掉落。

青竹灵境差异：

- 区域名：青竹灵境。
- Boss：竹灵守卫。
- 奖励偏向：丹 + 灵。
- 通关更容易获得灵草、回春丹，以及丹/灵倾向奖励。
- 难度高于青云山道，定位为入门功法后的第一道门槛。
- 宝物：灵竹玉佩。

解锁条件：

- 角色等级至少 LV5。
- 青云山道至少通关 1 次。
- 消耗 100 灵石。

### 5.1 灵竹玉佩

灵竹玉佩是青竹灵境的区域宝物。

效果：

- 普通历练开始时，20% 概率返还 1 点灵力。
- 只在普通历练开始时触发。
- 使用秘境令直接挑战 Boss 时不触发。
- 不影响 Boss 开战最低灵力需求。

提示文案：

```text
灵竹玉佩生辉
返还 1 点灵力
```

### 5.2 美术素材规划

V1.2 采用“最小新素材”策略。青竹灵境必须有独立第一视觉，但不为复用事件重新制作整套图标。

必需新增素材：

| 素材 | 用途 | 建议源路径 | 固件目标 |
|---|---|---|---|
| 青竹灵境背景 | 历练主场景背景 | `assets/raw/backgrounds/bamboo_realm_scene.png` | `firmware/ai_pet/assets/bamboo_realm_scene.h` |
| 竹灵守卫 Boss | 青竹灵境 Boss 待机/战斗显示 | `assets/raw/bosses/bamboo_guardian_2x2.png` | `firmware/ai_pet/assets/bamboo_guardian.h` |
| 灵竹玉佩图标 | 宝物页展示区域 2 宝物 | `assets/raw/items/spirit_bamboo_jade.png` | 可并入区域宝物图标头文件 |

可复用素材：

- 四类事件图标继续复用 V1.1 青云图标：灵草、妖兽、受伤修士、捷径。
- 宠物行走/站立素材继续复用现有青云宠物预览。
- 背包里的通用道具图标继续复用现有图标。
- 秘境令图标可沿用原青云信物图标，但文案必须改为“秘境令”。

青竹灵境背景交付标准：

- PNG，RGB 或 RGBA。
- 横向构图，比例接近 `128:54`。
- 推荐原图尺寸 `1280x540`，最低建议 `512x216`。
- 画面主题为青竹、灵雾、石径或竹林秘境。
- 不包含文字、UI、边框、水印、宠物、人物、Boss 或道具。
- 左侧或中间偏左留出宠物站位，右侧留出事件对象或 Boss 站位。
- 小屏下必须保持明暗层次，不可过亮导致浅色文字不可读。

竹灵守卫 Boss 交付标准：

- PNG 精灵表，`2x2` 网格，共 4 帧。
- 背景为纯洋红 `#FF00FF` 或透明背景。
- 3/4 侧面或正侧面，适合放在 128x160 小屏右侧。
- 主体为竹木、青玉、灵光构成的守卫或灵兽，不使用现代机械元素。
- 四帧主体身份、比例、底部锚点和配色保持一致。
- 主体不跨格，不接触边缘，不包含文字、UI 或地面背景。

灵竹玉佩图标交付标准：

- PNG，单个道具居中。
- 背景为纯洋红 `#FF00FF` 或透明背景。
- 外观为青竹纹玉佩、竹叶吊坠或青玉牌。
- 轮廓清晰，小屏下能与青云剑、秘境令区分。
- 不包含文字、数字、边框或 UI。

占位规则：

- 开发早期允许青竹灵境临时复用青云背景和青云妖狼 Boss 图，以保证规则和 UI 先编译通过。
- V1.2 发布验收前，青竹灵境背景和竹灵守卫 Boss 必须替换为独立素材。
- 灵竹玉佩若素材未完成，发布前至少需要一个可区分的简单图标或固件绘制占位，不能只显示文字。

转换流程：

- 背景素材应使用现有青云背景转换脚本扩展为多区域背景输入。
- Boss 素材应沿用现有首领精灵转换流程，输出 RGB565 像素数组和透明 mask。
- 道具图标应沿用现有 UI 图标转换流程，避免手写像素数组。

## 7. 战斗压力与区域难度

V1.2 提高 Boss 压力，但保留自动战斗，不新增主动技能。

青云山道调整：

- Boss 基础 HP：40 -> 60。
- Boss 基础伤害：10 -> 13。
- 体倾向减伤：`min(6, 体/5)` 改为 `min(5, 体/6)`。
- 前 10 轮 HP 成长：+20%/轮 -> +25%/轮。
- 前 10 轮伤害成长：+12%/轮 -> +15%/轮。
- 第 11 轮后 HP 成长：+5%/轮 -> +6%/轮。
- 第 11 轮后伤害成长：+3%/轮 -> +4%/轮。

后续区域难度更大：

| 区域 | 基础 HP | 基础伤害 | 前 10 轮 HP | 前 10 轮伤害 | 后续 HP | 后续伤害 |
|---|---:|---:|---:|---:|---:|---:|
| 青云山道 | 60 | 13 | +25% | +15% | +6% | +4% |
| 青竹灵境 | 90 | 17 | +27% | +17% | +7% | +5% |
| 云海剑台 | 125 | 22 | +30% | +19% | +8% | +6% |
| 丹霞药谷 | 165 | 28 | +33% | +22% | +9% | +7% |
| 玄河古战场 | 220 | 36 | +36% | +25% | +10% | +8% |

为了支撑后续区域，Boss HP 类型从 `uint8_t` 升级为 `uint16_t`：

- `PetSaveData::bossHp`
- `PetSaveData::bossMaxHp`
- `GameState::bossMaxHp()`
- Python 参考模型对应字段

UI 血条仍按百分比绘制，数字显示需要支持 3-4 位 HP。

## 8. 存档与兼容

V1.2 新增持久字段：

- 四门功法等级。
- Boss HP 由 8 位升级为 16 位。

因此必须提升 `kSaveVersion`。延续 V1.1 策略，不迁移旧结构；检测到旧版本或结构尺寸不匹配时初始化新存档。

功法等级建议使用固定数组：

```cpp
uint8_t techniqueLevels[4];
```

索引顺序与四维倾向一致：剑、丹、体、灵。

## 9. 实现边界

V1.2 包含：

- 四门功法等级、升级规则和被动效果。
- 状态页功法概览和功法详情/修炼页。
- 秘境令作为可消耗直达 Boss 资源。
- 青竹灵境开放。
- 区域 3-5 锁定预置。
- Boss HP `uint16_t` 化。
- 区域难度阶梯和奖励偏向。
- 青竹灵境背景、竹灵守卫 Boss、灵竹玉佩图标的素材规划与接入。
- 固件与 Python 参考模型同步。

V1.2 不包含：

- 主动技能。
- 装备栏。
- 稀有度、随机词条、强化、合成。
- 新 AI Hook 协议。
- 进化分支重做。
- 区域 3-5 的完整事件文案、美术和 Boss 独立机制。
- 为青竹灵境重新制作四类事件图标或宠物行走素材。
- 关机期间离线恢复补算。

## 10. 验证

测试需要覆盖：

- 功法升级成功、倾向不足、灵石不足、满级。
- 四门功法效果分别参与战斗、道具、恢复、灵力上限计算。
- 功法不影响进化分支。
- 九转丹法 Lv9 将被动恢复间隔改为 3 分钟，且不补算关机时间。
- 万灵息法 Lv9 提高灵力上限但不立即回满当前灵力。
- 秘境令入境前提示、消耗、拒绝使用、灵力不足不提示或不消耗。
- 受伤修士事件成功获得秘境令但不自动解锁 Boss。
- 青竹灵境解锁条件、进入、普通历练、Boss 战和通关奖励。
- 灵竹玉佩只在普通历练开始时 20% 概率返还灵力，秘境令直达 Boss 不触发。
- 青竹灵境背景、竹灵守卫 Boss、灵竹玉佩图标均可在固件中渲染，且发布前不再使用青云 Boss 作为最终占位。
- 青云山道 V1.2 难度数值。
- 区域 3-5 显示锁定且不可进入。
- Boss HP 超过 255 时 Python 与固件一致，UI 血条显示正常。
- `PetSaveData` 版本提升后旧存档失效并初始化新角色。
- 固件编译成功，Python 测试通过。
